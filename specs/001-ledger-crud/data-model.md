# Phase 1 Data Model: Basic CRUD for Ledger Services

**Feature**: `001-ledger-crud` · **Spec**: `./spec.md` · **Research**:
`./research.md` · **Date**: 2026-04-27

This document describes the durable schema (PostgreSQL), the Domain types
(C# `sealed class` + Builder), and the DTO shapes that cross layer
boundaries. It does NOT describe the gRPC wire format — that lives in
`./contracts/ledger.v1.proto`.

---

## 1. Aggregates and entities

### 1.1 `Ledger` (aggregate root)

| Field | Type | Notes |
|---|---|---|
| `Id` | `Guid` (UUIDv7) | Primary key, generated by the application at create time. |
| `OwnerId` | `Guid` | Stable user identifier from `ICurrentUser`. Indexed. |
| `Name` | `string` (1–100) | Trimmed; uniqueness enforced **case-insensitive per owner** at the DB level (see `idx_ledger_owner_name_lower`). |
| `Description` | `string?` (0–500) | Optional. |
| `CurrencyCode` | `string` (ISO 4217 alpha-3) | Validated by `[ValidIsoCurrency]`. |
| `Status` | `LedgerStatus` enum | `Active = 1`, `Archived = 2`. Default `Active` on create. |
| `Version` | `long` (≥ 1) | Monotonically incremented by every successful update; surfaces to clients as the opaque `version_token`. |
| `CreatedAt` | `DateTimeOffset` (UTC) | Set once on insert, never modified. |
| `LastModifiedAt` | `DateTimeOffset` (UTC) | Set on insert and on every update. |

**Invariants** (validated by `LedgerBuilder.Build()` →
`DomainValidator.Validate(ledger)`):

- `Name` is non-null, non-empty after trim, and ≤ 100 chars.
- `Description`, when present, is ≤ 500 chars.
- `CurrencyCode` is in the ISO 4217 alphabetic set
  (`ICurrencyCatalog.IsSupported`).
- `Version >= 1`.
- `CreatedAt <= LastModifiedAt`.

**State machine**:

```text
              Update(name|description)            Update(status=Active)
            ┌─────────────────────────────┐    ┌─────────────────────────┐
            │                             │    │                         │
            ▼                             │    │                         ▼
        ┌───────┐    Update(status=Archived)   ┌────────┐   (no other transitions allowed
        │Active │ ──────────────────────────▶  │Archived│   while archived; delete is rejected)
        └───────┘                              └────────┘
            │                                       │
            │ Delete (hard)                         │ Delete  → REJECTED (must un-archive first)
            ▼                                       └─────────▶ rejected
       ┌────────┐
       │ <gone> │   (audit row remains)
       └────────┘
```

Allowed transitions (see also FR-007a, FR-008, Acceptance Scenarios 3.4 / 3.5 / 4.4):

- `Active → Active` on attribute update (name, description, currency-not-changed).
- `Active → Archived` on `status=Archived` update.
- `Archived → Active` on `status=Active` update (un-archive). All other
  attribute changes on an `Archived` ledger are rejected with
  `Result.Failure(new Error("ledger.archived.read_only", ...,
  ErrorType.Validation))`.
- Hard-delete is allowed only from `Active`. From `Archived`, delete is
  rejected with `Result.Failure(new Error("ledger.archived.cannot_delete",
  ..., ErrorType.Validation))`.

### 1.2 `AuditEntry` (separate aggregate, write-only from this feature)

| Field | Type | Notes |
|---|---|---|
| `Id` | `long` (`bigserial`) | Sole purpose: deterministic ordering. |
| `ActorId` | `Guid` | From `ICurrentUser.UserId` at write time. |
| `LedgerId` | `Guid` | The affected ledger. |
| `EventType` | `AuditEventType` enum | `Create=1`, `Update=2`, `Delete=3`. |
| `EventAt` | `DateTimeOffset` (UTC) | Defaulted to `now()` in SQL. |
| `Payload` | `JsonDocument` | Snapshot of the post-event ledger row, or the pre-delete row for `Delete`. |

**Invariants**:

- Audit rows are append-only. There is no update or delete from
  application code.
- The retention purge job (`AuditRetentionPurgeService`) is the ONLY
  writer that issues `DELETE FROM ledger_audit` and only for rows older
  than 1 year.
- Audit rows are NOT exposed via any gRPC method or HTTP endpoint
  (FR-012). Read access is admin/ops via direct SQL.

## 2. PostgreSQL schema

```sql
-- src/Infrastructure.Migrations/Scripts/0001_Initial.sql (DbUp embedded resource)

CREATE TABLE ledger (
    id                uuid           PRIMARY KEY,
    owner_id          uuid           NOT NULL,
    name              varchar(100)   NOT NULL,
    description       varchar(500)   NULL,
    currency_code     char(3)        NOT NULL,
    status            smallint       NOT NULL DEFAULT 1,   -- 1=Active, 2=Archived
    version           bigint         NOT NULL DEFAULT 1,
    created_at        timestamptz    NOT NULL,
    last_modified_at  timestamptz    NOT NULL,
    CONSTRAINT ck_ledger_status CHECK (status IN (1, 2)),
    CONSTRAINT ck_ledger_version_positive CHECK (version >= 1),
    CONSTRAINT ck_ledger_currency_alpha3 CHECK (currency_code ~ '^[A-Z]{3}$')
);

-- Case-insensitive uniqueness of name within a single owner (FR-003).
CREATE UNIQUE INDEX ux_ledger_owner_name_lower
    ON ledger (owner_id, lower(name));

-- Listing: deterministic order keyed by (last_modified_at DESC, id DESC).
CREATE INDEX ix_ledger_owner_status_lastmodified
    ON ledger (owner_id, status, last_modified_at DESC, id DESC);

CREATE TABLE ledger_audit (
    id            bigserial     PRIMARY KEY,
    actor_id      uuid          NOT NULL,
    ledger_id     uuid          NOT NULL,
    event_type    smallint      NOT NULL,
    event_at      timestamptz   NOT NULL DEFAULT now(),
    payload       jsonb         NOT NULL,
    CONSTRAINT ck_audit_event_type CHECK (event_type IN (1, 2, 3))
);

-- Retention purge support.
CREATE INDEX ix_ledger_audit_event_at
    ON ledger_audit (event_at);

-- Admin lookups by ledger.
CREATE INDEX ix_ledger_audit_ledger_event_at
    ON ledger_audit (ledger_id, event_at DESC);
```

Migration project: `src/Infrastructure.Migrations/` — a Generic Host
console (`Microsoft.NET.Sdk` + `OutputType=Exe`) running a
`BackgroundService` that invokes DbUp against the Aspire-injected
`ConnectionStrings:ledger`, applies pending `Scripts/NNNN_*.sql`
embedded resources, then signals
`IHostApplicationLifetime.StopApplication`. Registered in the AppHost
as an init-container the Api waits on via `.WaitFor(migrator)` —
schema is therefore present before the first request lands. Scripts
are append-only: corrections ship as `0002_*.sql` and forward, never
by editing a committed script.

### 3.1 `Ledger` (shape)

```csharp
public sealed class Ledger : IValidatableObject
{
    [Required]                              public Guid           Id               { get; private init; }
    [Required]                              public Guid           OwnerId          { get; private init; }
    [Required, StringLength(100, MinimumLength = 1)]
                                            public string         Name             { get; private init; } = default!;
    [StringLength(500)]                     public string?        Description      { get; private init; }
    [Required, ValidIsoCurrency]            public string         CurrencyCode     { get; private init; } = default!;
                                            public LedgerStatus   Status           { get; private init; }
    [Range(1, long.MaxValue)]               public long           Version          { get; private init; }
                                            public DateTimeOffset CreatedAt        { get; private init; }
                                            public DateTimeOffset LastModifiedAt   { get; private init; }

    private Ledger() { }
    public static LedgerBuilder Builder() => new();

    public IEnumerable<ValidationResult> Validate(ValidationContext _)
    {
        if (string.IsNullOrWhiteSpace(Name))
            yield return new ValidationResult("Name must not be whitespace.", [nameof(Name)]);
        if (CreatedAt > LastModifiedAt)
            yield return new ValidationResult("LastModifiedAt must be >= CreatedAt.",
                                              [nameof(LastModifiedAt)]);
    }

    public sealed class LedgerBuilder
    {
        // private fields...
        public LedgerBuilder WithId(Guid id)                          { /* ... */ return this; }
        public LedgerBuilder WithOwnerId(Guid ownerId)                { /* ... */ return this; }
        public LedgerBuilder WithName(string name)                    { /* ... */ return this; }
        public LedgerBuilder WithDescription(string? description)     { /* ... */ return this; }
        public LedgerBuilder WithCurrencyCode(string code)            { /* ... */ return this; }
        public LedgerBuilder WithStatus(LedgerStatus status)          { /* ... */ return this; }
        public LedgerBuilder WithVersion(long version)                { /* ... */ return this; }
        public LedgerBuilder WithTimestamps(DateTimeOffset created,
                                            DateTimeOffset modified) { /* ... */ return this; }
        public Result<Ledger> Build() =>
            DomainValidator.Validate(new Ledger { /* assign fields */ });
    }
}
```

**Notes**:

- Constructor is private. The Builder is the only way to construct a
  `Ledger`.
- Properties are `private init` — initialised by the Builder and never
  mutated thereafter. Mutations go through new Builder calls in the
  Application handler (e.g., for an update).
- `[Required]`, `[StringLength]`, `[Range]`, `[ValidIsoCurrency]` are
  the declarative invariants; `IValidatableObject.Validate` covers the
  cross-property rule (`CreatedAt <= LastModifiedAt`) and the
  whitespace-only check for `Name`.

### 3.2 Canonical error codes (`LedgerErrors`)

| Constant | `ErrorType` | When |
|---|---|---|
| `Validation` | `Validation` | Generic — used by `DomainValidator` aggregation. |
| `NameAlreadyExists` | `Conflict` | Duplicate name within owner. |
| `NotFound` | `NotFound` | Owner mismatch or row absent (does not distinguish to caller — FR-009). |
| `Conflict` (version) | `Conflict` | Stale `version_token` on update. |
| `ArchivedReadOnly` | `Validation` | Attempt to mutate archived attributes. |
| `ArchivedCannotDelete` | `Validation` | Attempt to delete archived ledger. |

These map through `ResultToRpcExceptionMapper` to gRPC status codes per
the table in the constitution (Principle VI > API boundary translation).

All command/query records are decorated:

```csharp
public sealed record CreateLedgerCommand(
    [property: Required, StringLength(100, MinimumLength = 1)] string Name,
    [property: StringLength(500)] string? Description,
    [property: Required, ValidIsoCurrency] string CurrencyCode);

public sealed record UpdateLedgerCommand(
    [property: Required] Guid Id,
    [property: Required] byte[] VersionToken,
    [property: StringLength(100, MinimumLength = 1)] string? Name,
    [property: StringLength(500)] string? Description,
    LedgerStatus? Status);                              // null = no transition

public sealed record DeleteLedgerCommand(
    [property: Required] Guid Id,
    [property: Required] byte[] VersionToken);

public sealed record GetLedgerQuery(
    [property: Required] Guid Id);

public sealed record ListLedgersQuery(
    bool IncludeArchived,
    string? PageCursor,
    [property: Range(1, 200)] int PageSize = 50);
```

`UpdateLedgerCommand.VersionToken` is the opaque bytes of the
version on the wire; the handler decodes it to `long` and uses it as
the `WHERE version = $expected` predicate in the SQL `UPDATE`.

### 4.1 Mapping layer between protobuf and Application

For every gRPC method, a hand-written translation record (decorated
with the same Data Annotations) sits between the generated protobuf
message and the Application command/query. Example:

```csharp
// Application/Features/Ledgers/Commands/CreateLedger/CreateLedgerRequestMap.cs
internal sealed record CreateLedgerRequestMap(
    [property: Required, StringLength(100, MinimumLength = 1)] string Name,
    [property: StringLength(500)] string? Description,
    [property: Required, ValidIsoCurrency] string CurrencyCode)
{
    public static Result<CreateLedgerRequestMap> From(CreateLedgerRequest msg)
    {
        var dto = new CreateLedgerRequestMap(
            Name: msg.Name?.Trim() ?? string.Empty,
            Description: string.IsNullOrWhiteSpace(msg.Description) ? null : msg.Description.Trim(),
            CurrencyCode: (msg.CurrencyCode ?? string.Empty).Trim().ToUpperInvariant());
        return DomainValidator.Validate(dto);
    }

    public CreateLedgerCommand ToCommand() => new(Name, Description, CurrencyCode);
}
```

This keeps the protobuf-generated type's lack of attributes from being
a constitution violation (carve-out per Principle VI > Validation).

Repository contract (in `Application/Abstractions`):

```csharp
public interface ILedgerRepository
{
    Task<Result<Ledger>>            CreateAsync(Ledger ledger, AuditEntry audit, CancellationToken ct);
    Task<Result<Ledger>>            UpdateAsync(Ledger updated, long expectedVersion, AuditEntry audit, CancellationToken ct);
    Task<Result>                    DeleteAsync(Guid ledgerId, Guid ownerId, long expectedVersion, AuditEntry audit, CancellationToken ct);
    Task<Result<Ledger>>            GetByIdAsync(Guid ledgerId, Guid ownerId, CancellationToken ct);
    Task<Result<LedgerListPage>>    ListAsync(Guid ownerId, bool includeArchived, string? pageCursor, int pageSize, CancellationToken ct);
}
```

Caching decorator wraps the same interface; only `GetByIdAsync` and
`ListAsync` consult `IFusionCache`. Writes invalidate by tag (see
`research.md` §4).
